    GREEN phase — implement consumer.go to make all tests pass:

    Package: `package sqssink`
    Package doc comment: mirrors NATSSinkConsumer doc style — explain SQS is stateless HTTP (no reconnect needed), Deliver is synchronous (CHK-01), RetryScheduler handles retries (DLV-03).

    **sqsAPI interface** (enables testing without live AWS):
    ```go
    type sqsAPI interface {
        SendMessage(ctx context.Context, params *sqs.SendMessageInput, optFns ...func(*sqs.Options)) (*sqs.SendMessageOutput, error)
        GetQueueAttributes(ctx context.Context, params *sqs.GetQueueAttributesInput, optFns ...func(*sqs.Options)) (*sqs.GetQueueAttributesOutput, error)
    }
    ```

    **SQSSinkConsumer struct**:
    ```go
    var _ router.Consumer = (*SQSSinkConsumer)(nil)

    type SQSSinkConsumer struct {
        id       string
        client   sqsAPI
        queueURL string
        m        *observability.KaptantoMetrics
    }
    ```

    **NewSQSSinkConsumer(id string, cfg config.SQSSinkConfig) (*SQSSinkConsumer, error)**:
    1. Build AWS options slice with awsconfig.WithRegion(cfg.Region)
    2. If cfg.AccessKeyID != "" && cfg.SecretAccessKey != "", append awsconfig.WithCredentialsProvider(credentials.NewStaticCredentialsProvider(...))
    3. awsconfig.LoadDefaultConfig with 10-second context timeout
    4. sqs.NewFromConfig(awsCfg) → concrete *sqs.Client satisfies sqsAPI
    5. Call GetQueueAttributes with FifoQueue attribute, 10-second timeout context
    6. Return error on any GetQueueAttributes failure
    7. Return error if Attributes["FifoQueue"] != "true" — error must contain "not a FIFO queue" and the queue URL
    8. Return &SQSSinkConsumer{id: id, client: client, queueURL: cfg.QueueURL}

    **Deliver(ctx context.Context, entry eventlog.LogEntry) error**:
    1. MessageGroupId: `h := fnv.New64a(); h.Write(entry.Event.Key); groupID := fmt.Sprintf("%016x", h.Sum64())`
    2. MessageDeduplicationId: `sum := sha256.Sum256([]byte(entry.Event.IdempotencyKey)); dedupID := fmt.Sprintf("%x", sum)[:64]`
    3. data, err := json.Marshal(entry.Event) — return wrapped error on failure
    4. Record start := time.Now()
    5. Call client.SendMessage with QueueUrl, MessageBody, MessageGroupId, MessageDeduplicationId, and MessageAttributes["Kaptanto-Idempotency-Key"] (DataType="String", StringValue=raw IdempotencyKey)
    6. Observe latency: c.m.QueuePublishLatency.WithLabelValues("sqs").Observe(time.Since(start).Seconds()) — only if c.m != nil
    7. On error: increment QueuePublishErrors, return wrapped error
    8. On success: increment QueuePublishTotal, return nil

    **Ping() error**: GetQueueAttributes with ApproximateNumberOfMessages, 5-second timeout. Return wrapped error on failure, nil on success.

    **Close()**: no-op (SQS is stateless HTTP — no connection to close). Comment explaining why.

    **SetMetrics(m *observability.KaptantoMetrics)**: c.m = m

    **ID() string**: return c.id

    For tests, define a fakeSQSClient struct implementing sqsAPI with configurable SendMessage and GetQueueAttributes behaviors. Inject via a package-internal constructor or exported field (use unexported field + constructor for test file in same package).
